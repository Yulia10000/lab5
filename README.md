# Library Book Issue — Hexagonal Architecture

## 1. Опис use case

Система реалізує процес видачі книги студенту в бібліотеці.

Основний сценарій:

1. Студент обирає книгу.
2. Система перевіряє доступність книги.
3. Перевіряється, чи не перевищив студент допустиму кількість книг.
4. Якщо всі умови виконані — книга видається студенту.
5. Якщо ні — система повертає повідомлення про помилку.



## 2. Обрана архітектура

У проєкті використано підхід **Hexagonal Architecture (Ports and Adapters)**.



## 3. Обґрунтування вибору

Обрана архітектура дозволяє:

* відокремити бізнес-логіку від технічної реалізації;
* зробити систему незалежною від способу збереження даних;
* легко замінювати інфраструктурні компоненти (наприклад, базу даних);
* забезпечити чіткий розподіл відповідальностей між компонентами.



## 4. Структура проєкту

Проєкт поділений на такі модулі:

* **Domain/**
  Містить бізнес-логіку системи:

  * Book — модель книги
  * Student — модель студента

* **Application/**
  Містить сценарій використання:

  * IssueBookService — логіка видачі книги

* **Ports/**
  Інтерфейси для взаємодії із зовнішніми компонентами:

  * BookRepository
  * StudentRepository

* **Adapters/**
  Реалізації інтерфейсів:

  * InMemoryBookRepository
  * InMemoryStudentRepository

* **UI/**
  Точка входу в програму:

  * Main.java — консольний інтерфейс



## 5. Розподіл відповідальностей

* **Domain (бізнес-логіка):**
  Містить правила роботи системи:

  * перевірка доступності книги;
  * обмеження кількості книг у студента.

* **Application (use case):**
  Координує виконання сценарію видачі книги.

* **Ports (інтерфейси):**
  Визначають контракти для роботи з даними.

* **Adapters (інфраструктура):**
  Реалізують збереження даних (у пам’яті).

* **UI:**
  Забезпечує взаємодію з користувачем.



## 6. Бізнес-логіка

Бізнес-логіка реалізована у класах:

* Book — перевіряє, чи доступна книга;
* Student — перевіряє, чи не перевищено ліміт книг.

Основні правила:

* книга не видається, якщо немає доступних екземплярів;
* студент не може взяти більше 3 книг.



## 7. Інтерфейси / порти / адаптери

**Ports:**

* BookRepository
* StudentRepository

**Adapters:**

* InMemoryBookRepository
* InMemoryStudentRepository



## 8. Інструкція запуску

1. Встановити Java (JDK 8 або новіше)
2. Відкрити проєкт у IntelliJ IDEA або Eclipse
3. Запустити файл Main.java (папка UI)
4. У консолі з’явиться результат виконання програми



## 9. Висновок

У даному проєкті реалізовано сценарій видачі книги студенту з використанням сучасного архітектурного підходу Hexagonal Architecture.
Бізнес-логіка відокремлена від інфраструктури, що забезпечує гнучкість, масштабованість та зручність підтримки системи.
